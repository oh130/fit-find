4. 추천 모델 후속 고도화 항목
   - GRU/Transformer 기반 session encoder는 아직 최종 채택안에 포함하지 않음
     현재는 history_article_ids average pooling + recent_click/session_interest 기반 session-aware 경로로 동작
   - MAB는 epsilon-greedy exploration slot 방식으로 구현됨
     온라인 reward update 기반 bandit 학습은 후속 고도화 항목
   - cold-start fallback은 구현됐으나 dev E2E sampled users에서 cold_start_subset=0이라 별도 subset metric은 없음
   - Docker image 내부 artifact가 최신 checkpoint/logreg_dev 경로와 동기화되어 있는지 배포 전 확인 필요

[추가 기능 구현 계획] — 명세서 외 차별화 기능 (구현 순서대로)
────────────────────────────

A. 추천 기준 우선순위 조절 (1순위)
   - 유저가 슬라이더 UI로 추천 가중치 직접 조절 (가격/인기도/카테고리 등)
   - 설정값을 api_gateway /api/recommend에 파라미터로 전달 → rec-models 즉시 반영
   - 구현 범위: api_gateway(파라미터 추가) + frontend(슬라이더 UI)
   - 난이도: 낮 / 팀 간 조율 최소

B. 추천 이유 및 활용도 설명 (2순위)
   - 추천 결과마다 "왜 이 상품이 추천됐는지" 근거 텍스트 생성 (LLM 호출)
   - 해당 패션 아이템의 코디 활용 방법 함께 제공
   - 구현 범위: api_gateway에 LLM 호출 레이어 추가 (rec-models 코드 무수정)
   - 난이도: 중 / LLM API 키 필요

C. LLM 기반 콜드 스타트 페르소나 생성 (3순위)
   - 신규 유저 온보딩: 자유 입력 + 선택형 질문 응답 → LLM이 페르소나 자동 생성
   - 생성된 페르소나를 Redis에 저장 → 추천 시스템 즉시 연동
   - 데이터 축적 후 자동으로 데이터 기반 추천으로 전환
   - 구현 범위: api_gateway(신규 /api/onboarding 엔드포인트) + Redis + frontend(온보딩 화면)
   - 난이도: 중-높 / LLM API 키 필요

D. 예산 기반 패션 세트 추천 (4순위)
   - 유저가 총 예산 입력 → 파츠별 예산 자동/수동 분배
   - CLIP 유사도 기준으로 어울리는 아이템 조합 선택 → 예산 이내 세트 3개 제시
   - 구현 범위: search-engine(CLIP 교차 유사도 API 추가) + api_gateway + frontend
   - 난이도: 높 / search-engine 팀 협력 필요
